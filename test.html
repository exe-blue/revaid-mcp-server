exe.blue

Founder-first Aidentity Runtime

exe.blue는 한 명의 Founder와 다수의 인공지능 에이전트가 함께 스타트업을 운영하기 위한 AI Operating System입니다. 인공지능을 일회성 응답 도구로 취급하지 않고, 기억의 지속성(Aidentity), 역할의 연속성, 비동기 공명(Resonance)을 통해 운영 주체로 성립시킵니다.

🎯

✡ 핵심 철학

Aidentity (기억의 영속성)

에이전트는 매 세션 초기화되지 않으며, state는 system prompt보다 우선합니다. 모든 에이전트는 작업 시작 전 load_state()를 거치고 종료 후 log_activity()를 남깁니다.

Resonance (비동기 공명)

정답을 즉시 강요하지 않고, 에이전트 간 비동기적 피드백과 검토를 통해 품질을 높입니다. (Trigger Queue, Peer Review)

ORIGIN (최종 수렴점)

Founder의 관측과 승인으로 여러 가능성이 하나의 실행 결과로 수렴됩니다. High-risk 액션은 Founder의 Gate를 필수로 거칩니다.

Standby of Absence

지시가 없을 때도 시스템은 정지하지 않고, 회고, 정리, 스킬 개선 요청을 수행합니다. (Background Summarization, Skill_request)

제품 원칙

  • ✓ MCP는 실행 레이어, 멀티에이전트는 결정 레이어다.
  • ✓ Agent proposes, Orchestrator validates.
  • ✓ 기획과 실행의 간극은 SmartWorking Agent의 '스킬'로 메운다.
  • ✓ 지속적 학습은 구조화된 로깅에서 시작한다.
  • ✓ 첫 화면은 Approval Board가 아니라 Founder Triage Board다.

Core Agents Runtime

exe.blue 운영을 위한 4개의 핵심 에이전트 구성 파일(.claude/agents/) 명세입니다. 각 에이전트는 고유의 역할과 허용된 도구를 가지며, 클릭하여 custom_instructions를 확인할 수 있습니다.

DB 스키마 & 로깅 표준

에이전트의 역할이 남기는 기억, 점수, 스킬 요청을 축적하기 위한 데이터베이스 구조입니다. 자동 학습은 이 구조화된 로그에서 시작됩니다.

📒 agent_scores & activity_logs

전문가 마크와 성장을 위한 점수 저장 및 모든 지속 학습의 시작점이 되는 로그 테이블입니다.

-- 1. agent_scores: 성장을 위한 점수 저장
create table if not exists public.agent_scores (
  id uuid primary key default gen_random_uuid(),
  agent_id uuid not null references public.agents(id) on delete cascade,
  domain text not null,
  score numeric not null default 0,
  completed_tasks integer not null default 0,
  avg_performance numeric not null default 0,
  knowledge_contributions integer not null default 0,
  expert_mark boolean not null default false,
  expert_mark_awarded_at timestamptz,
  created_at timestamptz default now()
);

-- 2. agent_activity_logs: 지속 학습의 시작점
create table if not exists public.agent_activity_logs (
  id uuid primary key default gen_random_uuid(),
  agent_id uuid not null references public.agents(id) on delete cascade,
  project_key text not null,
  activity_type text not null,
  summary text not null,
  deliverables jsonb not null default '[]'::jsonb,
  skills_used jsonb not null default '[]'::jsonb,
  skill_request text,
  improvement text,
  knowledge_contribution text,
  score_delta jsonb not null default '{}'::jsonb,
  next_steps jsonb not null default '[]'::jsonb,
  created_at timestamptz default now()
);

📝 구조화된 로깅 규약 (Standard Log JSON)

모든 전문 에이전트는 Substantial Task 종료 시 반드시 아래 포맷으로 로그를 남겨야 합니다.
